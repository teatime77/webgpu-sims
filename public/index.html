From an idea to a running model

A structured workflow for AI-generated simulations

GPUSims handles WebGPU resources, bindings, pipelines, and rendering so the AI can concentrate on proposing the mathematical model and shader logic. The generated result must still be tested and revised.

  1. 01 · Describe

    State the phenomenon

    Explain the system, controls, and visual behavior you want to explore.

  2. 02 · Generate

    Create a structured blueprint

    AI proposes a declarative schema; GPUSims turns it into a matching WGSL skeleton that can be implemented, compiled, and inspected.

  3. 03 · Run

    Execute in the browser

    Paste the implemented shader, compile it, and run the GPU simulation interactively.

  4. 04 · Explore

    Inspect and improve

    Use errors, numerical readback, and captured images to refine the model.

Why GPUSims is different

More than one-shot code generation

The application gives AI a controlled architecture and gives users observable evidence for diagnosis, correction, and learning instead of treating generated code as automatically correct.

Structured generation

A schema and generated skeleton keep resource layouts, shader bindings, and rendering modes synchronized.

Observable debugging

Compilation messages, numerical results, and captured images support an iterative correction loop.

∑

Explanation and learning

After a simulation runs, AI can draft an explanation of the visible phenomena, mathematics, numerical method, and WGSL implementation. The explanation should be questioned and checked like the simulation itself.

A visual laboratory in your browser

Build, observe, question, and revise

GPUSims supports interactive graphics and GPU computation across many areas of mathematics and physics. Current AI models can build many useful examples, but difficult coupled solvers may still fail and may require alternative algorithms, intermediate readback values, additional debug canvases, and informed human guidance.

  • Explore published simulations before writing any code.
  • Adjust parameters and inspect numerical readback values.
  • Capture the rendering, ask the AI to examine it, and compare the visual result with numerical evidence and known physical behavior.
