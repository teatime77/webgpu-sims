Why It Works: The Power of Separation

This platform leverages WebGPU to deliver lightning-fast 3D animation and parallel computing directly in your browser. To make this accessible, we handle the heavy lifting: the app automatically manages all the complex WebGPU orchestration and boilerplate.

This strict separation means the AI is only responsible for creating the schema and writing the core logic. By removing the burden of GPU memory management, the AI can focus entirely on translating advanced mathematics into flawless code.

From One Prompt to an AI-Checked Simulation and Article

The local Python agent automates the complete development cycle by coordinating the AI API, Selenium, Chrome, and GPUSims. It is designed for advanced users who have configured Python, the required API credentials, and the app's local development environment.

  1. 1. Generate the simulation schema: The agent sends your simulation request together with the GPUSims architecture rules and saves the generated schema.
  2. 2. Build and implement the shaders: GPUSims converts the schema into WGSL skeletons, and the AI fills in the mathematical and physical logic.
  3. 3. Run the simulation automatically: Selenium enters the generated code into the app, compiles the WebGPU pipeline, and launches the simulation in Chrome.
  4. 4. Detect and repair errors: Compilation and runtime errors are returned to the AI. Revised schema or shader code is applied, and the simulation is run again.
  5. 5. Verify numerical and visual results: The agent checks GPU readback values when available, captures the rendered canvas, and asks the AI to evaluate both the mathematics and the image. Corrections are repeated when necessary.
  6. 6. Generate the learning materials: After the automated checks pass, the AI writes a Markdown article explaining the observable phenomena, mathematical background, and implementation. The agent also creates the thumbnail and article metadata.
The automated workflow produces a complete simulation package: schema, WGSL shaders, captured image, thumbnail, explanatory Markdown article, and article metadata.

Technical Specifications & Limitations

While WebGPU unlocks incredibly powerful parallel processing inside the browser, it has a few architectural differences compared to full-scale numerical computing systems like CUDA:

  • Precision: It currently supports single-precision floating-point arithmetic (f32), rather than double-precision.
  • Libraries: It cannot natively utilize traditional numerical computing libraries like LAPACK.

The Silver Lining: WebGPU and CUDA share fundamental principles regarding parallel thread execution and memory optimization. Mastering simulations here serves as an excellent, visual stepping stone if you decide to explore CUDA in the future.
