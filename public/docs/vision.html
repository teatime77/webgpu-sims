Project vision

Learn through executable experiments

GPUSims explores how structured human–AI conversations, executable simulations, numerical observations, and visual evidence can support learning—and whether the resulting experience can eventually improve AI systems.

Motivation

From equations to observable behavior

Advanced physical theories are often introduced through dense notation and long derivations. A simulation adds another path: propose a model, change a parameter, observe the response, inspect the numerical state, and return to the equations with a concrete phenomenon or failure in mind.

GPUSims does not remove the need for mathematics or domain expertise. It creates an inspectable bridge between symbolic ideas, numerical algorithms, GPU implementation, visible behavior, and the conversation used to diagnose the result.

Visual intuition

Fields, waves, particles, surfaces, and phase changes become systems that can be manipulated, measured, and challenged rather than diagrams that remain fixed on a page.

Progressive explanation

AI can draft an initial explanation, while the user can ask increasingly focused questions, propose alternative algorithms, and request intermediate values or debug views when the first result is inadequate.

Human–AI collaboration

A structured conversation with executable feedback

Ordinary code generation ends when text is produced. GPUSims continues the process by compiling the generated code and producing evidence: errors, numerical values, and images. The user can then turn that evidence into a substantive conversation rather than a mechanical request for replacement code.

  1. Hypothesis
  2. Execution
  3. Observation
  4. Correction

The user can question each stage. Was the schema appropriate? Would another numerical method be more stable? Which intermediate quantity would distinguish competing explanations? Are conserved quantities stable? Does the rendered geometry match the model? These interventions may improve the performance of the human–AI system even when the underlying model weights do not change.

Practical experience

What made the collaboration effective

While creating many different simulations through extended dialogues with AI, I found that successful results rarely came from a single prompt. Progress usually depended on an iterative collaboration in which the user helped frame the problem, interpret the evidence, and decide what to try next.

Several practices proved especially effective:

  • Starting with a simple model and increasing its complexity gradually, rather than attempting the most difficult version immediately.
  • Reconsidering or replacing the numerical algorithm when the first approach was unstable, inaccurate, or poorly suited to the problem.
  • Exposing intermediate calculations through numerical readback values or additional debug canvases, so that hidden failures could be examined directly.
  • Specifying algorithms, boundary conditions, parameter ranges, and validation criteria in enough detail to give the AI a well-defined problem.
  • Trying a different AI model when repeated attempts with the same model did not produce a reliable solution.

This experience suggests that the user should understand enough of the simulation to question assumptions, request meaningful diagnostics, and judge whether a proposed correction addresses the actual problem. Simply delegating the entire task to the AI can conceal plausible-looking errors and deprive the conversation of the evidence needed for improvement.

The relevant unit of capability may therefore be the collaborative system formed by the user, the AI, the simulation environment, and the accumulated conversation. In this respect, the process resembles working with a human colleague: progress comes from proposing ideas, examining intermediate work, challenging explanations, dividing responsibilities, and revising decisions in response to evidence. This is different from treating learning only as the unilateral delivery of large amounts of data to a model. Structured collaboration may also produce higher-quality experience records that could later support external memory, evaluation, or training.

Research question

Can AI learn through simulation?

Large language models learn powerful patterns from text, images, video, and code, but a simulation offers an active loop. Within a conversation, an AI can propose a model, execute it, observe numerical and visual consequences, discuss alternative explanations, revise the implementation, and repeat the experiment. The resulting improvement may come from in-context adaptation, account-level memory, accumulated human guidance, or external experience records rather than from changes to the model’s parameters.

This is a research direction, not a completed claim. GPUSims currently provides an inference-time experimental environment; it does not itself update model weights. The open questions are whether issue-focused simulation conversations produce transferable improvements, whether successful and failed trajectories can become persistent training data or external memory, and whether a less capable model can improve by working with a stronger model and executable feedback.

Scope

Accessible experimentation, not automatic truth

Beautiful output can still be wrong, and some requests remain beyond current models. In my tests, neither Gemini 3.1 Pro nor ChatGPT successfully completed this task: “Create a two-dimensional simulation of incompressible fluid flowing past a circular obstacle. Visualize the velocity or vorticity field and show the formation of the wake.” This negative result is important: a large gallery of successful simulations does not demonstrate general scientific competence.

The project therefore emphasizes inspection, discussion, and explicit failure analysis. Users should compare results with theory, known limiting cases, conservation laws, convergence tests, independent calculations, and expert review. Failed attempts should record the hypotheses considered, diagnostics requested, alternative algorithms proposed, and evidence that changed the conclusion.

Long-term direction

A browser-based laboratory for more people

The long-term goal is to let learners, teachers, programmers, and researchers construct inspectable mathematical experiments while also building a reusable record of scientific problem solving. A research system could compare a fresh model with one given prior simulation conversations, retrieve lessons from related failures, or let a stronger model critique a weaker model. Measurable improvement on unfamiliar tasks—not merely a successful conversation—would be the evidence needed to claim learning. GPUSims is intended as a platform for exploring that question, not as proof that it has already been answered.
