Introduction

Making advanced simulation more accessible

WebGPU, the successor to WebGL, greatly expands the browser’s support for GPGPU—general-purpose computation on graphics processing units.

Using these capabilities, I developed an application that makes it easy to create mathematical and physical simulations with the help of AI.

AI can generate code for complex simulations covering subjects ranging from electromagnetism, fluid dynamics, and quantum mechanics to particle physics. It can even reproduce profound physical ideas—such as the Higgs mechanism, in which spontaneous symmetry breaking gives mass to gauge bosons—inside a tiny 32 × 32 two-dimensional universe.

The central idea

With vibe coding, even people without advanced programming or physics expertise can experiment with sophisticated simulations.

The application

A browser-based environment for AI-generated simulations

GPUSims is a web application available at gpusims.com.

Click Explore Simulations to browse existing simulations. Click Start Building Now to begin creating your own.

You can use the application without signing in. Publishing a simulation, however, requires you to sign in with a Google account.

Architecture

How the simulation pipeline works

GPUSims divides the development process into a declarative schema, an application-generated WGSL skeleton, AI-generated simulation logic, and browser-based execution.

1. The schema

The application uses JSON-like data called a schema to declare GPU resources, shaders, user-interface controls, and other elements of a simulation.

The following is a simple schema example. Because the schema is generated by AI, users do not need to understand every detail.

TypeScript · Simulation schema
const schema: SimulationSchema = {
    name: "Cumulative Monte Carlo Pi Estimation",
    resources: {
        Params: {
            type: 'uniform', obj: state
        },
        Points: {
            type: 'storage', format: 'f32', topology: 'point-list'
        },
    },

    shaders: [
        {
            id: 'generate_and_count',
            type: 'compute',
            workgroupSize: WORKGROUP_SIZE, workgroupCount: DISPATCH_X,
            bindings: [
                { resource: 'Params', varName: 'params' },
                { resource: 'Points', access: 'read_write' },
            ]
        },
    ],
};

2. The shader skeleton

The application generates a shader-code skeleton from the schema created by the AI. In WebGPU, programs that run on the GPU are written in WGSL, a language with syntax somewhat similar to Rust.

In this example, the application generates the buffer declarations and the header of the main function. The shader implementation is also written and debugged by AI, so users do not necessarily need to understand the WGSL code.

WGSL · Generated skeleton
@group(0) @binding(0) var<uniform> params: ParamsStruct;
@group(0) @binding(1) var<storage, read_write> Points: array<f32>;
@group(0) @binding(2) var<storage, read_write> WorkgroupSums: array<f32>;

@compute @workgroup_size(64, 1, 1)
fn main(@builtin(global_invocation_id) id: vec3<u32>) {
    // TODO: Write compute logic for generate_and_count
}

3. Implementing the shader

The generated skeleton is sent back to the AI, which implements the mathematical and physical logic inside the main function.

4. Running the simulation

The application combines the schema with the completed shader and runs the simulation.

  • Drag with the left mouse button to rotate the 3D view.
  • Drag with the right mouse button to move the viewpoint.

Design principle

Why this approach works

WebGPU is a powerful but highly complex API. Asking an AI to generate every part of a complete WebGPU program places a heavy burden on the model.

GPUSims handles the repetitive and error-prone WebGPU setup. The AI can therefore concentrate on the parts where it provides the most value:

  • Formulating the mathematical model
  • Translating equations into GPU code
  • Checking numerical and visual results
  • Diagnosing and correcting errors
Separation of responsibilities

The app manages WebGPU resources, pipelines, rendering, and boilerplate. AI focuses on translating mathematics and physics into executable simulation logic.

Guided process

The complete workflow

The overall process for creating and running a simulation is shown below.

  1. Give the system prompt to the AI

    Click Copy AI System Prompt in GPUSims. Paste the copied prompt into the AI chat so that the model understands the schema format, resource layouts, shader bindings, and output rules.

  2. Ask the AI to create a schema

    Describe the simulation you want. For advanced scientific topics, an English prompt may work better because much of the relevant technical literature and source code is written in English.

  3. Generate the WGSL skeleton

    Paste the AI-generated schema into GPUSims and click Copy WGSL Skeleton. Then paste the skeleton into the AI chat and ask the model to implement it.

  4. Run the simulation

    Paste the completed WGSL code into GPUSims and click Run Simulation.

A typical schema request is extremely simple:

Prompt
Create a schema for an N-body simulation.

Debugging and verification

Running successfully is not the same as being correct

Error handling

If the schema or shader contains an error, the application displays an error screen.

Click Copy Error Message, paste the error into the AI chat, and ask the model to fix it.

Error-correction prompt
Fix this error.

Expected ':' but found '.' at index 319

The AI examines the error, identifies its likely cause, and returns a corrected schema or shader. Paste the revised code back into GPUSims and continue the workflow.

Verifying numerical results

Even when a simulation appears to run correctly, I recommend asking the AI to verify its output. A visually impressive animation is not necessarily a physically correct simulation.

The upper-right corner of the simulation screen displays parameter values and calculated results. Click the copy iconto copy those values, paste them into the AI chat, and ask whether they are mathematically and physically reasonable.

Verifying the 3D visualization

Click Capture to save the current simulation view as an image. Upload that image to the AI and ask:

Visual-verification prompt
Is this correct?

It is surprisingly common to be delighted by a beautiful 3D animation, only for the AI to point out that it is physically incorrect. When this happens, the AI can investigate the likely cause and generate a revised schema or shader.

Important limitation

AI-assisted verification is useful for iteration, but it is not formal scientific validation. Complex or high-stakes models still require knowledgeable human review.

Using an additional debug canvas

For a complex simulation, displaying intermediate results on an additional canvas can make debugging much easier. For example:

Debug-canvas prompt
Please create a schema that allows intermediate calculation results
to be output to an additional canvas.

The AI can then generate a schema containing the configuration for a separate debugging canvas. Incidentally, I often use Google Translate when converting my own prompts into English.

Understanding the result

Ask the AI to write an explanatory article

Even after a simulation is running correctly, I often do not understand what is happening internally. In that case, I ask the AI to write an explanatory article:

Article-generation prompt
Please write an article explaining the phenomena observable in this
simulation, the underlying mathematics, and the code implementation.

To be honest, I often still do not fully understand the first explanation. However, I can repeatedly ask more detailed questions about the parts I find difficult. By continuing this process, I can gradually develop at least a basic understanding of the theory and implementation.

Automation

Is the human actually doing anything? Why not use an agent?

Apart from the initial request to create the schema, the user mainly performs simple actions: copying and pasting code, copying error messages, clicking buttons, and uploading screenshots for verification.

I have confirmed that this process can be automated. A Python program can use Selenium to operate the browser application and an official AI API to conduct the conversation. An agent can therefore automate the following cycle:

  • Create the schema and shader
  • Run the simulation
  • Detect compilation and runtime errors
  • Debug the generated code
  • Inspect numerical values and captured images
  • Generate an explanatory article and metadata

In my tests, the gemini-3.1-pro-preview model available through my current Gemini subscription could automate only relatively simple simulations successfully.

Use official APIs for automation

Many AI services prohibit using browser-automation tools such as Selenium to enter prompts directly into their web-based chat interfaces. Automated systems should generally use an official API instead.

I plan to publish code for automatically creating and running simulations with an AI agent in the near future.

Education

Learning mathematics and physics through simulation

Traditionally, learning mathematics and physics has meant struggling through difficult textbooks. Yet 3D animations of electromagnetism, fluid mechanics, quantum mechanics, and other subjects can be extraordinarily beautiful.

Until recently, directly experiencing the beauty of advanced mathematical and physical theories was largely limited to highly gifted students, researchers, and specialists. AI-generated 3D simulations may make these worlds accessible to almost anyone.

I hope GPUSims encourages more people to become interested in mathematics and physics.

Future concept

Can AI learn through simulation?

This section describes a future idea rather than a completed feature.

Current large language models learn sophisticated patterns of reasoning from enormous collections of text. However, they do not possess direct physical experience. They may construct a linguistic “world model,” but they often lack an intuitive understanding of how physical systems behave.

In GPUSims, an AI can propose a model, generate and run a simulation, observe images and numerical data, verify the output, modify the implementation, and repeat the experiment.

  1. Hypothesis
  2. Execution
  3. Observation
  4. Correction

I am interested in whether repeating this active exploratory loop could help an AI construct a more grounded world model based on physical laws rather than language alone.

Technical scope

Limitations of WebGPU

Numerical computation with WebGPU has several important limitations:

  • Standard WebGPU does not support 64-bit double-precision floating-point arithmetic.
  • Established numerical libraries such as LAPACK cannot be used directly.

Nevertheless, WebGPU and CUDA share many fundamental concepts related to parallel computation and memory optimization. Experience with GPU numerical computation in GPUSims may therefore provide useful preparation for learning CUDA in the future.

Conclusion

A new way to explore mathematical worlds

GPUSims is an experiment in making advanced simulation more approachable. The application handles the machinery of browser-based GPU computation while AI concentrates on modeling, implementation, debugging, and explanation.
